Опис проєкту remapper

Цей проєкт призначений для обробки Excel-файлів з наказами про втрати майна,
виділення потрібних записів, перенесення їх у проміжну SQLite-базу та
автоматичного заповнення підсумкового журналу втрат по окремих аркушах.

Основна ідея

Код бере вихідний Excel-файл з наказами, аналізує рядки з датами, номерами
наказів і сумами по різних службах/напрямах, визначає до якого аркуша журналу
належить кожен запис, а потім переносить ці дані у підготовлений файл журналу.

Основні задачі, які виконує код

1. Підготовка та нормалізація Excel-даних
   - оновлює заголовки у вихідному Excel-файлі;
   - перевіряє коректність дат;
   - готує структуру таблиці до подальшої обробки.

2. Зчитування наказів про втрати з Excel
   - проходить по рядках аркуша `Sheet1`;
   - читає дату наказу, номер наказу та суми по категоріях;
   - пропускає порожні або невалідні рядки.

3. Визначення відповідного журналу для запису
   - використовує мапінг `department_to_sheet`;
   - розподіляє записи по аркушах журналу: `БпЛА`, `ОВТ`, `ЗВ`, `ЗББ`,
     `ЗУ`, `РЕЧ`, `НСО (БТ)`, `Ел-тех`, `ІС`, `ГЕО`, `прод`, `пмм`,
     `СВТ (АС)`, `мед`, `КЕС( СІ-ІЗ)`, `Метрологія`.

4. Обробка змінених або скасованих записів
   - аналізує закреслені рядки в Excel;
   - шукає первинний наказ, до якого відноситься зміна;
   - зберігає інформацію про пов'язані (вкладені) накази.

5. Збереження результату в SQLite
   - записує проміжні дані в базу `wasted.db`;
   - зберігає номер наказу, дату, підрозділ, аркуш журналу, суму,
     номер рядка та ознаку заповнення;
   - використовує таблицю `wasted` як центральне сховище для подальшого
     заповнення журналу.

6. Автоматичне заповнення вихідного журналу втрат
   - відкриває файл журналу;
   - знаходить перший вільний рядок на кожному аркуші;
   - переносить туди дату, номер наказу та суму;
   - окремо додає пов'язані зміни/уточнення червоним шрифтом.

7. Валідація та контроль якості
   - перевіряє, чи вже занесені записи присутні у базі;
   - підсвічує проблемні рядки кольором;
   - відмічає випадки, коли запис не знайдено, знайдено кілька збігів
     або є проблема з датою.

8. Допоміжна робота зі стилями Excel
   - визначає закреслення тексту, колір тексту та колір фону ячейки;
   - використовується для аналізу форматування і службової перевірки.

Ключові файли

- `core.py` — спільні константи, перевірка дат, мапінг підрозділів на аркуші.
- `prepare_excel.py` — підготовка структури вихідного Excel-файлу.
- `extract_orders.py` — витягує накази з Excel і записує їх у `wasted.db`.
- `fill_journal.py` — заповнює підсумковий журнал на основі даних з бази.
- `detect_filled_orders.py` — визначає вже внесені записи та позначає їх.
- `validate.py` — перевіряє коректність дат і структуру записів у журналі.
- `styles.py` — кольори і підсвітка проблемних рядків.
- `xls_styles.py` — читання форматування окремих ячейок Excel.

Залежності

- `openpyxl` — робота з Excel-файлами;
- `pandas` — допоміжне читання табличних даних.

Короткий підсумок

Проєкт автоматизує перенесення даних з наказів про втрати майна у книгу
обліку/журнал, зменшує ручну роботу, допомагає розкладати записи по потрібних
аркушах і перевіряти коректність уже внесених даних.
